# Eorzea Weekly — Shared Newsletter Configuration

This file contains shared rules and preferences that apply to ALL newsletters generated by the scheduled tasks. Topic-specific configs (e.g., `ffxiv.md`) inherit and extend these rules.

---

## Date & Recency Rules

These rules are **non-negotiable** and override everything else:

1. **Before doing anything**, run `date` in bash to get today's exact date.
2. The valid content window is **8 days back from today** through today.
3. **Every** piece of content in the newsletter MUST have a verifiable publication date within this window.
4. You have **ZERO reliable knowledge** about any topic. Do NOT use training data for topic-specific content. Every fact, name, date, and detail MUST come from a web search performed during THIS run.
5. After finding each piece of content, **verify its publication date** by clicking through to the source. If you cannot confirm it was published within the last 8 days, **discard it**.
6. If a section has no real news this week, write "Quiet week on this front — see you next Saturday!" or omit the section entirely. **NEVER fabricate or guess.**

## Search Strategy

For each search query:
- Always include the **current month and year** to bias toward recent results (e.g., "FFXIV news March 2026")
- Run **multiple targeted searches** rather than one broad query
- Click through to verify dates and extract real details
- Gather: headlines, summaries (2-3 sentences), source URLs, image URLs where available
- Prefer official sources first, then major outlets, then community sources

## Writing Style

- Tone: Warm, enthusiastic, slightly irreverent. Like a friend who's really into the topic catching you up over coffee.
- Use **bold** for the single most important phrase in the intro paragraph.
- Summaries should be 2-3 sentences max — enough to understand the story, short enough to want to click through.
- Always link to the original source so the reader can go deeper.
- If a quote is included, it must be a real, attributed quote sourced this week.

## Image Handling

- When a real image URL is found (from official sites, Reddit/imgur, YouTube thumbnails), use an `<img>` tag.
- When no image is available, use the emoji placeholder div from the template.
- Never hotlink from sites known to block it. Prefer: Lodestone, imgur, YouTube thumbnail URLs (`https://img.youtube.com/vi/VIDEO_ID/hqdefault.jpg`).

## Output

- Save the newsletter HTML to the output path specified in the topic config.
- Also save a dated archive copy: `{output_path}/{newsletter-name}-{YYYY-MM-DD}.html`

## Final Verification Checklist

Before finishing, confirm:
- [ ] Every piece of content has a verifiable source from the last 8 days
- [ ] No content was generated from training data or memory
- [ ] All links point to real, working URLs
- [ ] The date shown is today's date
- [ ] The HTML is well-formed (no broken tags)
